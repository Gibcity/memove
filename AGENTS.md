## 1. What this is

Self-hosted, real-time collaborative travel planner. NestJS + React 19 + Vite +
better-sqlite3 + MCP server. Single pnpm workspace, three packages, one shared
schema package. AGPL-3.0.

reads. Drift here multiplies across every future session.

## 3. Setup commands

From `memove/` root:

```bash
pnpm install                            # workspace install
pnpm --filter @memove/shared build      # MANDATORY before client/server dev
pnpm dev                                # concurrently runs shared watch + server + client
```

Server listens on `:3000`, client on `:5173` (Vite default). SQLite lands at
`server/data/<env>.db` (gitignored).

**One-liner for first-time setup:**
```bash
pnpm install && pnpm --filter @memove/shared build && pnpm dev
```

## 5. Code style ŌĆö the non-negotiables

1. **TypeScript strict mode** in every package. `noUncheckedIndexedAccess`,
   `exactOptionalPropertyTypes` are on.
2. **No `any`.** Use `unknown` + Zod parse, or define a type.
3. **No cross-package imports** between `client/src/` and `server/src/`. The
   seam is `@memove/shared` (Zod schemas) + the server's OpenAPI spec.
4. **Conventional commits** enforced by CI. Format: `type(scope): message`.
   Scopes mirror the domain (`feat(relocation):`, `fix(maps):`).
5. **Match existing style.** No reformatting unrelated lines. No "while I'm here" cleanups.
6. **Tests ship with the change.** New service ŌåÆ new test file. No exceptions.
7. **80%+ coverage** on the changed code path.

## 6. Domain conventions (the parts that bite)

### 6.1 Add-ons
New top-level features go through `isAddonEnabled(ADDON_IDS.X)`. The `relocation`
add-on is the canonical example ŌĆö mirror its structure (controller, service,
schema, register in `addons.ts`).

### 6.2 Auth / guard order
Every controller uses `@UseGuards(JwtAuthGuard)` + `@UseInterceptors(IdempotencyInterceptor)`
on writes. Re-use `JwtAuthGuard`, do not roll your own. Passkey/WebAuthn handled
by existing endpoints ŌĆö extend, don't duplicate.

### 6.3 Database
- `better-sqlite3` everywhere. Synchronous API by design.
- One module owns its tables (e.g. `trips/` owns `trips`, `days`, `packing_items`).
- Migrations live in `server/src/<domain>/migrations/`, run in
  `database/runMigrations()` at boot. New table ŌåÆ new migration, no exceptions.

### 6.4 Realtime
WebSocket gateway is one per domain (`TripsGateway`, `CollabGateway`). Client
subscribes via `useWebSocket(domain)`. Don't open raw sockets from React.

### 6.5 Schema-first
**Define the contract in `shared/src/<domain>/<domain>.schema.ts` before
writing server or client code.** Server validates with `.parse()`, client
imports the inferred type. OpenAPI is generated from the server controller
decorators ŌĆö keep them in sync with the Zod schema.

### 6.6 Errors
Server throws typed exceptions from `@nestjs/common` (`NotFoundException`,
`BadRequestException`, etc.). No raw `throw new Error('something')`.
Client surfaces errors via the toast store ŌĆö never `console.error` in production code.

## 9. For subagents: how to return work

Every delegation must end with this confirmation block (see root
`CURRENT-WORK.md` for context):

```
### CONFIRMATION
- Commit: [hash or "not committed"]
- Files changed: [exact paths, one per line]
- TSC: [pass/fail + error count]
- Tests: [N pass / N fail or "not run"]
- Skipped: [what was skipped and why, or "none"]
- Push: [yes/no]
```

**Do not edit `CURRENT-WORK.md`** ŌĆö only the orchestrator updates it.
**Do not push without `git status --short` showing a clean tree** other than
your intended changes.
